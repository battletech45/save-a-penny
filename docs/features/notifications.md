# Notifications

## Overview

Notifications expose user-facing status changes and alerts. They are user-scoped and support read/unread tracking.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/v1/notifications` | Create a notification |
| GET | `/api/v1/notifications` | List notifications (paginated) |
| GET | `/api/v1/notifications/{id}` | Get notification details |
| PUT | `/api/v1/notifications/{id}` | Update a notification |
| DELETE | `/api/v1/notifications/{id}` | Delete a notification |
| GET | `/api/v1/notifications/unread-count` | Get unread count |
| PATCH | `/api/v1/notifications/mark-all-read` | Mark all as read |

## Notification Types

| Type | Source | Description |
|------|--------|-------------|
| `GOAL_OFF_TRACK` | Goal scheduler | A goal is falling behind its target |
| Various others | System events | Generated by the backend |

## Fields

| Field | Notes |
|-------|-------|
| `title` | Short summary |
| `message` | Detailed description |
| `type` | Notification category |
| `read` | Boolean, defaults to `false` |
| `metadata` | Optional JSON payload with context |

## Unread Count

```bash
curl -X GET "http://localhost:8080/api/v1/notifications/unread-count" \
  -H "Authorization: Bearer <accessToken>"
```

Returns:

```json
{
  "unreadCount": 3
}
```

## Mark All Read

```bash
curl -X PATCH "http://localhost:8080/api/v1/notifications/mark-all-read" \
  -H "Authorization: Bearer <accessToken>"
```

Sets all unread notifications for the current user to `read = true`.
